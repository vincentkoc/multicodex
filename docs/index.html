self-contained room control

docs for the room that runs on your machine.

MultiCodex is a host-local control room for normal local Codex sessions. The browser gives the host a live lane view, visible coordination actions, and lane previews without turning the product into remote-workspace mush.

host-local by default Room server, conductor, and state stay on the host machine.
preview aware Attach a per-lane browser preview and keep it visible in the room.
policy scoped Participant policy controls what the host can suggest, steer, or interrupt.
quickstart

start a room

Run the doctor check, then start a host from the repository root.

npx --yes @vincentkoc/multicodex@latest doctor
npx --yes @vincentkoc/multicodex@latest host --repo . --title "OpenAI event build"
participant

join a lane

Use the invite command from the room. The join command launches the normal local Codex TUI on the participant machine. 

npx --yes @vincentkoc/multicodex@latest join '<invite-url>' \
  --repo . \
  --name Queenie \
  --policy suggest \
  --terminal-mirror \
  --terminal-control
`--terminal-control` only works when the invite and participant both opt in. Running the same command again resumes the lane instead of creating drift.
preview

preview docs or app work

Serve the local docs page, then attach the URL to the lane preview pane. This is the cleanest way to make docs reviewable inside MultiCodex.

serve the docs

Use the repo script to expose docs/ on http://127.0.0.1:4173.

attach the preview

Point the lane at that URL with multicodex preview set or pass --preview-url on join. 

pnpm docs:serve
npx --yes @vincentkoc/multicodex@latest preview set "$MULTICODEX_LANE_URL" \
  --url http://127.0.0.1:4173
host controls

what the host can do

  • create, recopy, and revoke named single-use invites;
  • remove a participant lane and revoke its capability;
  • inspect messages, plans, commands, approvals, and turn state;
  • send visible suggestions and follow-up actions;
  • steer or interrupt active turns when the lane policy allows it.
security

security model

  • host, invite, and lane capabilities are distinct;
  • room snapshots never expose the capability URLs;
  • the participant Codex app-server binds to loopback only;
  • terminal mirror data stays bounded and ephemeral;
  • repository contents, credentials, and hidden reasoning stay private.
repo dev

develop from the repo

Use the local CLI directly when you want to work on the implementation.

pnpm install
pnpm multicodex doctor
pnpm multicodex host --repo .
pnpm check

Preview flow

  • run pnpm docs:serve
  • open the lane preview pane
  • attach http://127.0.0.1:4173
  • use the docs page as the visual target
